## 💡 Tips for Using This Documentation

### For Students:
1. **Don't read everything at once** - Start with QUICK_START.md
2. **Bookmark MENTOR_CHEATSHEET.md** - Search it when stuck
3. **Read FORK_WORKFLOW.md gradually** - Come back to sections as needed
4. **Keep WORKFLOW_SUMMARY.md open** - Quick reference while coding

### For Mentor:
1. **Print MENTOR_CHEATSHEET.md** - Have it handy during sprint
2. **Update docs as you go** - Add Q&A that come up
3. **Point students to specific sections** - Don't say "read the docs", say "check MENTOR_CHEATSHEET.md → 'I have merge conflicts!'"
4. **Use docs in code reviews** - Link to relevant sections
